Getting Started

TB Registry is the package registry for the ToolBox Framework. Install and manage modules using the tb CLI with the CloudM module manager.

pip install toolboxv2

Module Management

Interactive Manager

The easiest way to manage modules is the interactive manager:

tb -c CloudM mods manager

Install a Module

tb -c CloudM mods install <module-name>

Or use the shortcut:

tb --install <module-name>

List Modules

tb -c CloudM mods list

Update Modules

tb --update <module-name>

Remove a Module

tb --remove <module-name>

Registry CLI

Direct access to the registry API for searching, downloading, and publishing.

Search & Browse

tb registry search <query>
tb registry list --type mod --sort downloads
tb registry info CloudM --versions

Download

tb registry download CloudM --version 2.0.0

Publishing

Publish your own modules to share with the community. You need to be a registered publisher.

1. Login

tb registry login

2. Generate Config

Create a tbConfig.yaml for your module:

tb -c CloudM mods gen-config <module-name>

3. Create Package

Register your package in the registry:

tb registry publish ./my-mod --create --metadata metadata.json

4. Upload Version

tb registry publish ./my-mod --upload --metadata metadata.json

5. Publish via Manager

Or use the interactive manager:

tb -c CloudM mods manager

Then select REGISTRY → Publish to registry

Set Visibility

tb registry publish my-mod --visibility public

Note: Public visibility requires a verified publisher. Use unlisted for testing.

Artifacts

Download official ToolBox applications and tools.

SimpleCore Desktop

The official ToolBox desktop application with GUI.

  • Windows (x64): .exe, .msi
  • macOS: .dmg (ARM64, x64)
  • Linux: .deb, .rpm, .AppImage

TB CLI

Command-line interface for ToolBox. Install via pip:

pip install toolboxv2

Upload Artifact Builds (Publishers)

curl -X POST https://registry.simplecore.app/api/v1/artifacts/SimpleCore/builds \
  -H "Authorization: Bearer $TOKEN" \
  -F "version=1.0.0" \
  -F "platform=windows" \
  -F "architecture=x64" \
  -F "installer_type=msi" \
  -F "file=@SimpleCore-1.0.0-win-x64.msi"

Admin

Admin tools for managing publishers and the registry.

Server-Side Admin CLI

Run directly on the registry server (no auth required):

python admin_cli.py --db ./data/registry.db

Use to create the first admin, manage publishers, and run SQL queries.

Remote Admin (via CLI)

tb registry admin publisher list --status pending
tb registry admin publisher verify --target <publisher-id>

API Reference

The registry provides a REST API for programmatic access. Base URL: https://registry.simplecore.app/api/v1

Authentication

  • GET /api/v1/auth/me — Current user info
  • GET /api/v1/auth/publisher — Publisher profile
  • POST /api/v1/auth/register-publisher — Register as publisher

Packages

  • GET /api/v1/packages — List all packages
  • GET /api/v1/packages/{name} — Get package details
  • POST /api/v1/packages — Create package (auth)
  • PATCH /api/v1/packages/{name} — Update package (auth)
  • DELETE /api/v1/packages/{name} — Delete package (auth)
  • POST /api/v1/packages/{name}/upload — Upload version (auth)
  • GET /api/v1/packages/{name}/versions/{v}/download — Get download URL
  • POST /api/v1/packages/{name}/versions/{v}/yank — Yank version (auth)

Artifacts

  • GET /api/v1/artifacts — List artifacts
  • GET /api/v1/artifacts/{name} — Get artifact details
  • POST /api/v1/artifacts — Create artifact (auth)
  • POST /api/v1/artifacts/{name}/builds — Upload build (auth)
  • GET /api/v1/artifacts/{name}/latest — Latest for platform
  • GET /api/v1/artifacts/{name}/versions/{v}/download — Download build

Publishers

  • GET /api/v1/publishers — List publishers
  • GET /api/v1/publishers/{slug} — Get publisher
  • POST /api/v1/publishers/verify — Submit verification (auth)

Search, Resolve & Versions

  • GET /api/v1/search?q={query} — Search packages
  • GET /api/v1/search/suggest?q={prefix} — Autocomplete
  • POST /api/v1/resolve — Resolve dependencies
  • GET /api/v1/resolve/check — Compatibility check
  • GET /api/v1/versions?names=a&names=b — Batch version query

Diff (Incremental Updates)

  • GET /api/v1/packages/{name}/diff/{from}/{to} — Diff info
  • GET /api/v1/packages/{name}/diff/{from}/{to}/download — Download patch
  • POST /api/v1/packages/{name}/diff/create — Pre-generate diff

Admin

  • GET /api/v1/packages/admin/pending — Pending publishers
  • POST /api/v1/packages/admin/{id}/verify — Verify publisher
  • POST /api/v1/packages/admin/{id}/reject — Reject publisher
  • POST /api/v1/packages/admin/{id}/revoke — Revoke verification

Example
